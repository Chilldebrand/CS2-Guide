# Boulder utility visual assets

Asset names use `<map>-<side>-<grenade>-<target>-<stage>`, where `<stage>` is `start`, `aim`, or `result`.

Each card must preserve the exact target, timing, purpose, and fallback from `maps/boulder/utility.md`. Until verified current-build geometry exists, keep Start, Aim, throw type, landing, and grenade-behavior details explicitly pending rather than inferred.

Remote images stay source-linked when reuse permission or current geometry cannot be verified. Do not create local lineup screenshots or claim fixed mechanics until a reliable post-update source supports them.
